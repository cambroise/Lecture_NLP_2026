From Transformer Families to BERT

What BERT is, why it matters, and how it works mathematically

Learning goals

By the end of this morning, you should be able to:

  • distinguish encoder-only, decoder-only, and encoder-decoder Transformer models
  • explain what BERT is and why it is suited for language understanding tasks
  • describe the mathematical flow of a BERT forward pass
  • connect the theory to a practical notebook on sentiment analysis

Audience already knows attention, so the presentation emphasizes architecture choices, representation learning, and the classification pipeline rather than re-deriving attention from first principles.

Three Transformer families

Family Core idea Typical direction Best for Example models
Encoder-only build contextual representations of the input bidirectional understanding, classification, tagging, retrieval BERT, RoBERTa, DistilBERT
Decoder-only predict the next token autoregressively left-to-right generation, dialogue, code, drafting GPT, Llama, Mistral
Encoder-decoder encode input, then generate output conditioned on it input-to-output translation, summarization, rewriting T5, BART, FLAN-T5

Key contrast

  • BERT asks: What does this sentence mean?
  • GPT asks: What token should come next?
  • T5/BART ask: Given this input, what output sequence should I generate?

Easy way to remember

  • BERT = understand text
  • GPT = generate text
  • Encoder-decoder = convert one text into another

Real-life uses of each family (1)

Encoder-only: understanding text

  • sentiment analysis for product reviews or movie reviews
  • email triage: complaint, refund, sales lead, urgent issue
  • resume screening: extract skills, roles, experience entities
  • search/ranking: compare query meaning with document meaning
  • fraud/compliance flags in chat logs or claims text

Real-life uses of each family (2)

Decoder-only: generating text

  • customer-support assistants
  • code completion and code explanation
  • report drafting from bullet points
  • brainstorming, tutoring, dialogue systems
  • synthetic text generation for prototyping

Real-life uses of each family (3)

Encoder-decoder: transforming text

  • translation
  • summarization of long documents
  • question answering from a passage
  • rewriting legal or technical text into simpler language
  • converting structured inputs into natural language reports

Why BERT is the focus today (1)

BERT is a strong choice when the goal is to understand a sentence rather than generate one.

Typical course examples:

  • Is a review positive or negative?
  • Which tokens are named entities?
  • Are two sentences paraphrases?
  • Which passage is most relevant to a search query?

Why BERT is the focus today (2)

Intuition

BERT sees both left and right context, so the meaning of a word depends on the full sentence.

  • “I went to the bank to deposit money.”
  • “I sat by the bank of the river.”

The token “bank” receives different contextual representations in the two sentences.

What BERT is

BERT = Bidirectional Encoder Representations from Transformers

In one sentence

BERT is a stack of Transformer encoder blocks that turns a tokenized sentence into context-aware vectors, then uses those vectors for downstream tasks.

Important properties

  • encoder-only architecture
  • bidirectional self-attention
  • trained first by pre-training, then adapted by fine-tuning
  • especially effective for classification and representation learning

BERT input representation

For a tokenized input of length \(n\), BERT builds one vector per token:

\[ x_i = e_i^{token} + e_i^{position} + e_i^{segment} \]

where:

  • \(e_i^{token}\) = token embedding
  • \(e_i^{position}\) = position embedding
  • \(e_i^{segment}\) = sentence-A / sentence-B embedding

Special tokens

  • [CLS] at the beginning: used as a sentence-level summary for classification
  • [SEP] to separate segments or mark sentence end

Matrix form: \(X^{(0)} \in \mathbb{R}^{n \times d}\), where \(d\) is the hidden size.

BERT encoder block: high-level view

  1. multi-head self-attention over all tokens
  2. residual connection + layer normalization
  3. position-wise feed-forward network
  4. residual connection + layer normalization

Repeated for \(L\) layers:

\[ X^{(0)} \rightarrow X^{(1)} \rightarrow \dots \rightarrow X^{(L)} \]

In bert-base-uncased:

  • \(L = 12\) layers
  • hidden size \(d = 768\)
  • \(12\) attention heads

BERT mathematically: self-attention step

Starting from a layer input \(X \in \mathbb{R}^{n \times d}\):

\[ Q = XW_Q, \qquad K = XW_K, \qquad V = XW_V \]

Scaled dot-product attention for one head:

\[ A = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right) \quad \quad Z = AV \]

Interpretation

  • each token computes how much it should attend to every other token
  • because BERT is bidirectional, a token can attend to both left and right context
  • padding positions are suppressed using an attention mask

BERT mathematically: multi-head + feed-forward

For multiple heads:

\[ \text{MultiHead}(X) = \text{Concat}(Z_1, \dots, Z_h)W_O \]

Then residual + normalization:

\[ \widetilde{X} = \text{LayerNorm}(X + \text{MultiHead}(X)) \]

Then the position-wise feed-forward network:

\[ \text{FFN}(x) = W_2\,\sigma(W_1x + b_1) + b_2 \]

and again:

\[ X' = \text{LayerNorm}(\widetilde{X} + \text{FFN}(\widetilde{X})) \]

Result: After the final layer, each token has a contextual embedding.

From BERT representations to classification (1)

For sentence classification, we use the final hidden state of [CLS]:

\[ h_{CLS} = X^{(L)}_{0,:} \]

Then a linear classifier predicts the label:

\[ \text{logits} = Wh_{CLS} + b \]

\[ p(y \mid x) = \text{softmax}(\text{logits}) \]

Training minimizes cross-entropy:

\[ L = -\log p(y_{true}\mid x) \]

From BERT representations to classification (2)

Example task

  • input: a movie review sentence
  • output: positive or negative sentiment

How BERT learns before fine-tuning

BERT is not trained from scratch on your task.

It is first pre-trained on very large corpora using masked language modeling.

Idea

Randomly mask some tokens and predict them from both left and right context.

\[ L_{MLM} = - \sum_{i \in M} \log p(x_i \mid x_{\setminus M}) \]

Why this matters? Pre-training teaches BERT general language structure:

  • syntax
  • semantic similarity
  • word sense from context
  • phrase-level and sentence-level regularities

Feature extraction vs full fine-tuning

Regime 1: feature extraction

  • keep BERT frozen
  • extract \(h_{CLS}\)
  • train a lightweight classifier on top

\[ \hat{y} = \text{Classifier}(h_{CLS}) \]

Regime 2: full fine-tuning

  • update all BERT parameters on the downstream task
  • the encoder itself becomes task-aware

\[ \hat{y} = \text{Classifier}(\text{BERT}(x)) \]

Trade-off. 1) frozen = simpler, cheaper, fast baseline. 2) fine-tuned = usually better performance

Practical case study: sentiment analysis notebook

A natural practical sequence for a notebook is:

  1. load a sentiment dataset such as SST-2
  2. tokenize sentences with a BERT tokenizer
  3. create input IDs and attention masks
  4. try frozen BERT + linear classifier
  5. try full fine-tuning with a classification head
  6. evaluate on validation data
  7. inspect errors and optional attention maps

Conceptual checkpoints

  • tokenization is not just preprocessing; it defines the model’s input units
  • [CLS] is used as a sentence representation for classification
  • the attention mask prevents padding from polluting representations
  • frozen features test the quality of pre-training alone
  • fine-tuning shows how task supervision reshapes the representation space

Typical discussion question

Why does full fine-tuning usually beat frozen features for sentiment?

Because the encoder can adapt its internal representation to sentiment-bearing patterns such as negation, intensifiers, sarcasm cues, and domain-specific wording.

Main takeaways

  • Transformer families solve different kinds of problems
  • BERT is encoder-only and built for understanding text
  • mathematically, BERT is repeated self-attention + FFN + residual + normalization
  • for classification, the final [CLS] representation is mapped to logits
  • in practice, BERT is powerful because pre-training + fine-tuning transfers language knowledge to real tasks
